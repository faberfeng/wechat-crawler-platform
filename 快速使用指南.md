## 📚 历史文章抓取功能

新版历史抓取脚本已开发完成！

### 使用方法

```bash
# 抓取"上海静安"公众号的历史文章
cd ~/Desktop/wechat-crawler-platform
python3 crawl_history_v2.py --account-id 1
```

### 功能特性

✅ **自动滚动加载** - 自动滚动页面加载所有文章
✅ **智能提取** - 支持多种页面结构
✅ **调试模式** - 可显示浏览器窗口便于调试
✅ **自动保存** - 保存到配置文件供定时使用
✅ **页面截图** - 保存页面截图和源码

### 参数说明

```
--account-id 1       # 公众号 ID（必需）
--debug              # 调试模式，显示浏览器窗口
--no-save            # 不保存到文件
```

### 文件输出

抓取后会生成以下文件：

1. **配置文件** - `config/article_urls.txt`
   - 供定时任务使用
   - 最多保存 50 篇文章（避免太长）

2. **批量文件** - `config/article_urls_{ID}_{时间}.txt`
   - 包含所有文章链接
   - 便于手动使用

3. **调试文件**（调试模式下）
   - `/tmp/wechat_crawl_screenshot_{biz}.png` - 页面截图
   - `/tmp/wechat_page_source_{biz}.html` - 页面源码

## 📊 工作流程

### 完整流程

```
1. 使用历史抓取脚本获取所有文章链接
   │
   ▼
2. 文章链接保存到 config/article_urls.txt
   │
   ▼
3. 定时任务每天 7:00 读取配置
   │
   ▼
4. 抓取每篇文章并保存到数据库
   │
   ▼
5. 等待第二天 7:00 继续抓取新文章
```

### 示例命令

```bash
# 第一步：抓取历史文章
python3 crawl_history_v2.py --account-id 1

# 第二步：查看配置文件
cat config/article_urls.txt

# 第三步：手动测试抓取
python3 crawl_articles.py --account-id 1

# 第四步：配置定时任务
crontab ~/Desktop/wechat-crawler-platform/crontab_7am.conf

# 第五步：等待自动运行（明天早上 7:00）
```

## 🛠️ 常见操作

### 修改运行时间

```bash

# 修改第一个数字（分钟）和第二个数字（小时）

2=https://mp.weixin.qq.com/s/文章3,https://mp.weixin.qq.com/s/文章4
```

### 手动抓取单篇文章

```bash
cd ~/Desktop/wechat-crawler-platform

# 使用 API
curl -X POST http://localhost:8002/api/v1/crawl/wechat \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -d '{"url":"https://mp.weixin.qq.com/s/文章链接","user_id":1,"account_id":1}'
```
