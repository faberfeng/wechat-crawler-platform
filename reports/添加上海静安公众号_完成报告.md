## ⏰ 定时抓取配置

### 当前系统状态

系统已支持定时任务功能，但需要手动配置。以下是几种配置方案：

### 方案一：使用系统的定时任务 API（推荐）

后端已实现任务调度服务（`app/services/scheduler.py`），可以创建定时抓取任务。

#### API 示例

```bash
# 1. 创建定时任务
curl -X POST http://localhost:8002/api/v1/tasks \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -d '{
    "task_type": "crawl_account",
    "account_id": 1,
    "schedule": "0 */2 * * *",
    "enabled": true
  }'

# 参数说明：
# account_id: 公众号 ID（1 = 上海静安）
# schedule: Cron 表达式
#   - "0 */2 * * *" = 每 2 小时执行一次
#   - "0 9,18 * * *" = 每天 9:00 和 18:00 执行
#   - "0 9 * * *" = 每天 9:00 执行
```

### 方案二：使用系统 Cron 定时任务

如果 API 不可用，可以直接配置系统 Cron：

```bash
# 编辑 crontab
crontab -e

# 添加定时任务（每2小时抓取一次）
0 */2 * * * cd ~/Desktop/wechat-crawler-platform/backend && python3 scripts/crawl_account.py 1 >> logs/cron.log 2>&1

# 或者每天早上 9 点抓取一次
0 9 * * * cd ~/Desktop/wechat-crawler-platform/backend && python3 scripts/crawl_account.py 1 >> logs/cron.log 2>&1
```

### 方案三：手动触发抓取

```bash
# 使用 API 触发
curl -X POST http://localhost:8002/api/v1/crawl/url \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -d '{
    "url": "https://mp.weixin.qq.com/s/文章链接",
    "user_id": 1
  }'
```

## 💡 重要提示

### Biz 参数说明

`__biz` 是微信公众号的唯一标识符：

- **格式**：Base64 编码字符串
- **示例**：`MzA5MzI3NjkyNA==`
- **解码后**：`3093276924`（数字ID）
- **用途**：
  - 访问公众号历史文章：`https://mp.weixin.qq.com/mp/profile_ext?__biz=...`
  - API 调用和文章抓取
  - 多公众号管理

### 定时抓取的必要性

微信公众号文章：
- ✅ 时效性强，需要及时跟踪
- ✅ 内容量大，需要定期抓取
- ✅ 有历史价值，需要归档保存
- ✅ 支持内容分析和二次创作

### 技术挑战

1. **反爬机制**：微信公众号有较强的反爬措施
2. **登录态管理**：需要维护有效的登录 Cookie
3. **增量抓取**：避免重复抓取已保存的文章
4. **稳定性**：确保长时间运行不崩溃
